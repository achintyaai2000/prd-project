Community Notes

A crowd sourced context layer that lets eligible contributors add factual notes to potentially misleading posts, visible to everyone once broadly agreed helpful across diverse viewpoints

Section 02

Executive Summary

Users often encounter viral posts that are misleading, lack context, or contain outdated information. Traditional fact checking methods struggle to scale to the volume and speed of content shared on the platform. Rapid spread of misinformation, platform trust issues, scaling limitations of human moderation, and the need for additional context make this a critical problem.

This PRD proposes Community Notes: a feature that allows eligible users to add contextual notes to posts that may be misleading, incomplete, or lacking important information. If enough contributors with diverse viewpoints agree that a note is helpful, it becomes visible to everyone.

Key capabilities (v1)

  • Eligibility based note writing (users meeting certain criteria)
  • Note rating system with diverse viewpoint requirements
  • Note visibility when broadly agreed as helpful
  • Note display on posts as context cards
  • Basic note categories: misleading, incomplete, lacks context, etc.
  • Contributor reputation tracking

Section 03

Problem Statement

The Core Problem: Users frequently encounter viral posts that are misleading, lack context, or contain outdated information. Traditional fact checking methods cannot scale to the volume and speed of content on the platform.

Who Has This Problem?

  • News consumers who cannot easily determine whether a viral claim is trustworthy
  • Casual social media users who don't have time to fact check every post they encounter
  • Researchers & journalists for whom viral posts often lack source attribution or context
  • Platform moderators who need scalable ways to surface contextual information

Evidence of the Problem

SignalData Point
Speed of misinformation spreadFalse or misleading posts can reach millions within hours; official fact checks often arrive too late
Platform trust issuesUsers increasingly question reliability of information on social media, reducing engagement and credibility
Scaling problemHuman moderation teams cannot review every post; professional fact checkers are limited in number
Context gapsMany posts are technically correct but omit critical context, not every misleading post is outright false

What Happens Without Community Notes?

  • Users face greater exposure to misleading information and have more difficulty distinguishing facts from rumors
  • Platform faces increased criticism, greater moderation burden, and lower user trust
  • Advertisers face concerns about brand safety and potential reputational risks

Section 07

User Journey & User Stories

Primary User Journey - Note Contributor

  1. User encounters a post that lacks context or appears misleading
  2. User taps "Add Note", writes context or correction, and selects a note category
  3. Note is submitted and enters the rating queue
  4. Contributors with diverse viewpoints rate the note
  5. Once enough contributors agree it is helpful, the note becomes visible to all users

Primary User Journey - Note Viewer

  1. User encounters a post with a visible Community Note
  2. User reads the context card to understand the correction or missing information
  3. User makes an informed decision, whether to share, engage, or skip

Section 08

Functional Requirements

FR-01  P0

Eligibility Requirements for Note Writing

Users must meet eligibility criteria to write notes (e.g. account age, activity level, no past violations). The system must verify eligibility before allowing note writing.

FR-02  P0

Note Writing Interface

Eligible users must be able to write notes (text input), select a note category (misleading, incomplete, lacks context, etc.), and submit notes for rating. The interface must be simple and intuitive, completable in ≤ 3 taps.

FR-03  P0

Note Rating System

Contributors must be able to rate notes as helpful or not helpful. The system must require ratings from contributors with diverse viewpoints before determining visibility to prevent ideological or coordinated bias.

FR-04  P0

Note Visibility Logic

Notes become visible only when enough contributors rate them helpful AND those contributors include diverse viewpoints. The system must calculate visibility automatically without manual intervention.

FR-05  P0

Note Display on Posts

Visible notes must appear on posts as context cards. Notes must load within ≤ 2 seconds and must be visible before the share button to ensure users see context before amplifying a post.

FR-06  P0

Cross Platform Note Support

Notes must work on Mobile (Android, iOS) and Web with a consistent UX across all platforms. No critical functionality should be missing on any platform.

FR-07  P0

Contributor Reputation Tracking

The system must track contributor reputation based on note helpfulness ratings and past contribution quality. This data informs eligibility determination and rating weight calculations.

FR-08  P0

Note Event Tracking

The system must track: notes written, notes rated, and notes visible. This data supports analytics, product optimisation, and stakeholder reporting.

Section 11

Assumptions & Constraints

Assumptions:

  • Users will write honest, helpful notes in good faith
  • Contributors will rate notes fairly and without coordinated bias
  • The diverse viewpoint requirement is sufficient to reduce ideological bias

Constraints:

  • Eligibility requirements will reduce the available contributor pool
  • Not all posts will have notes, coverage will be incomplete at launch
  • Brigading and coordinated manipulation are inherent challenges to address continuously
  • A speed vs. accuracy trade-off exists: faster visibility may increase bias risk
